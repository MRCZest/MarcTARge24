Waterfall

Waterfall is the oldest and most criticized process model. It was described by Winston W. Royce in 1970. The principle of this model is that activities take place sequentially in separate stages. Royce divided the stages as follows:

Process Stages

  • Requirement Definition

    Divided into two parts: system analysis (everything surrounding the specific software) and requirements analysis. The behavior, performance, interfaces, etc., of the system are documented.

  • System and Software Design

    Focuses on key program features like data structures, software architecture, interface specifications, and procedural and algorithmic details. The project's quality can be assessed. The result is documented.

  • Implementation and Module Testing

    The system described in the project is programmed as a set of modules and programs, which are tested separately. The more detailed the project, the easier and more mechanical the implementation stage can be.

  • Integration and System Testing

    Programs and modules are integrated and tested as a whole system. After testing, the product is delivered to the client. The testing focuses on logical details and whether the system meets functional requirements.

  • Usage and Maintenance

    This is usually the longest phase. The system is modified as users discover bugs, the environment and working conditions change, or the client requires new functionality. This phase repeats all previous phases in the context of changing an existing system.
